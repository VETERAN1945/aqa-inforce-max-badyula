MANUAL TEST CASES - Room Booking
=================================

Test Environment: https://automationintesting.online/
Browser: Chrome


TC_001: Book room with valid data
----------------------------------
Type: Positive

Precondition:
- User is on https://automationintesting.online/
- Double Room has at least 2 consecutive available days within the next 2 weeks

Steps:
1. Click "Book now" on Double Room
2. Select any 2 consecutive available dates
3. Fill in form:
   - Firstname: John
   - Lastname: Smith
   - Email: john.smith@test.com
   - Phone: 38050123456789
4. Click "Reserve Now"

Expected:
- Booking confirmation is displayed
- Selected dates are shown in confirmation

TC_003: Unavailable date selection causes crash
------------------------------------------------
Type: Negative / Bug Report

Precondition:
- User is on https://automationintesting.online/
- Double Room was already booked in this session,
  making at least one date shown as "Unavailable"

Steps:
1. Click "Book now" on Double Room
2. Click on any date marked as "Unavailable"

Expected:
- Date should not be selectable
- OR system shows error: "This date is unavailable"

Actual:
- APPLICATION CRASHES
- Shows: "Application error: a client-side exception has occurred"
- User must refresh page to continue

Status: CRITICAL BUG

TC_007: All fields empty validation
-------------------------------------

- Double Room has at least 2 consecutive available days within the next 2 weeks

Steps:
1. Click "Book now" on Double Room
2. Select any 2 consecutive available dates
3. Leave ALL fields empty:
   - Firstname: (empty)
   - Lastname: (empty)
   - Email: (empty)

Expected:

  "Firstname should not be blank"
  "Lastname should not be blank"
  "must be a well-formed email address"
  "must not be empty"

- Lastname: 3-30 chars
- Phone: 11-21 chars
